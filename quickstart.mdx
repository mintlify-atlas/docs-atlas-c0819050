The primary way to use the Ruby Advisory Database is through [bundler-audit](https://github.com/rubysec/bundler-audit), a command-line tool that checks your `Gemfile.lock` against the database and reports any gems with known security vulnerabilities.

<Steps>
  <Step title="Install bundler-audit">
    Install the gem globally so it's available in any project:

    ```bash
    gem install bundler-audit
    ```
  </Step>

  <Step title="Update the database">
    Fetch the latest advisories from the Ruby Advisory Database:

    ```bash
    bundle-audit update
    ```

    <Tip>
      `bundle-audit update` clones or pulls from the [ruby-advisory-db repository](https://github.com/rubysec/ruby-advisory-db) on GitHub. Run it regularly to ensure you're checking against the most recent advisories.
    </Tip>
  </Step>

  <Step title="Scan your project">
    Run the check from within your project directory (where your `Gemfile.lock` lives):

    ```bash
    bundle-audit check
    ```
  </Step>

  <Step title="Interpret the results">
    When bundler-audit finds a vulnerability, the output looks like this:

    ```
    Name: actionpack
    Version: 7.0.4.0
    Advisory: CVE-2023-22795
    Criticality: Unknown
    URL: https://github.com/rails/rails/releases/tag/v7.0.4.1
    Title: ReDoS based DoS vulnerability in Action Dispatch
    Solution: upgrade to ~> 5.2.8, ~> 6.1.7.1, >= 7.0.4.1

    Vulnerabilities found!
    ```

    Each finding shows you:

    - **Name / Version** — the affected gem and the version in your `Gemfile.lock`
    - **Advisory** — the CVE or GHSA identifier
    - **URL** — a link to the full security disclosure
    - **Title** — a short description of the vulnerability
    - **Solution** — the version range you need to upgrade to

    If no vulnerabilities are found, the output ends with `No vulnerabilities found`.
  </Step>
</Steps>

## Using bundler-audit in CI/CD

Add a vulnerability scan to your CI pipeline so every pull request is checked automatically. Here is an example GitHub Actions workflow:

```yaml
name: Security audit

on:
  push:
    branches: [main]
  pull_request:

jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Ruby
        uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.3'
          bundler-cache: true

      - name: Install bundler-audit
        run: gem install bundler-audit

      - name: Update advisory database
        run: bundle-audit update

      - name: Audit dependencies
        run: bundle-audit check
```

<Warning>
  If your project uses [Rails LTS](https://railslts.com/), its backported security patches are not tracked in the Ruby Advisory Database. Add the relevant CVEs to the `ignore:` list in your `.bundler-audit.yml` to avoid false positives. See the [Rails LTS CVE list](https://makandracards.com/railslts/474590-list-cves-addresses-rails-lts) for the full set of identifiers to ignore.
</Warning>
